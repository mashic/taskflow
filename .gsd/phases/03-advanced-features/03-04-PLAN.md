<objective>
Implement team invitations using the Factory pattern to create email or shareable link invitations.

Purpose: Enable board collaboration with role-based team membership, demonstrate Factory pattern
Output: Invitation and team modules with Factory pattern for invitation creation

**Design Pattern: Factory Pattern (GoF)**
- **Definition:** Define an interface for creating an object, but let subclasses decide which class to instantiate. Factory Method lets a class defer instantiation to subclasses.
- **Implementation:** InvitationFactory with create(type, data) method that returns different invitation configurations based on InvitationType (EMAIL or LINK). Email invitations are single-use with 7-day expiry; link invitations are multi-use with 30-day expiry.
- **Interview talking point:** "I used the Factory pattern for invitations because email and link invitations have different creation logic - different expiration times, usage limits, and token lengths. The factory encapsulates this logic and makes it easy to add new invitation types."
</objective>
