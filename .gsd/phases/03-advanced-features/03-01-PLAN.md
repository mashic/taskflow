<objective>
Implement task comments with @mentions and real-time notifications using the Strategy pattern.

Purpose: Enable collaboration through task discussions and demonstrate Strategy pattern for interview
Output: Full-stack comment system with notification strategies (in-app, WebSocket)

**Design Pattern: Strategy Pattern (GoF)**
- **Definition:** Define a family of algorithms, encapsulate each one, and make them interchangeable. Strategy lets the algorithm vary independently from clients that use it.
- **Implementation:** NotificationStrategy interface with InAppNotificationStrategy and WebSocketNotificationStrategy implementations. CommentsService iterates over all strategies when a mention is detected.
- **Interview talking point:** "I used the Strategy pattern for notifications so we can easily add new channels (email, push) without modifying the comment logic. Each strategy implements a common interface, and the service iterates through all registered strategies."
</objective>
