<objective>
Implement activity logging using the Decorator pattern to automatically log all board changes.

Purpose: Provide audit trail and activity feed for collaboration, demonstrate Decorator pattern
Output: Activity module with decorator-based logging and frontend feed component

**Design Pattern: Decorator Pattern (GoF)**
- **Definition:** Attach additional responsibilities to an object dynamically. Decorators provide a flexible alternative to subclassing for extending functionality.
- **Implementation:** @AuditLog() method decorator with AuditLogInterceptor that wraps controller methods. The interceptor reads metadata and logs to ActivityService after successful operations.
- **Interview talking point:** "I used the Decorator pattern via NestJS interceptors to add audit logging without modifying existing controller logic. The @AuditLog decorator marks methods for logging, and the interceptor handles the cross-cutting concern."
</objective>

  <action>
    1. Create AuditLog decorator:
       - audit-log.decorator.ts: SetMetadata decorator
       - Interface AuditLogOptions: { action: string, entity: string }
       - Export AUDIT_LOG_KEY constant
    
    2. Create AuditLogInterceptor:
       - Inject ActivityService and Reflector
       - Read AUDIT_LOG_KEY metadata from handler
       - If no metadata, pass through without logging
       - After successful response, log activity with:
         - userId from request.user.sub
         - action and entity from metadata
         - entityId from result.id or request.params.id
         - boardId from result.boardId or request.params.boardId or request.body.boardId
    
    3. Create ActivityRepository:
       - create(data): Activity
       - findByBoardId(boardId, limit?): Activity[] with user relation, ordered by createdAt desc
       - findByEntityId(entityType, entityId): Activity[]
    
    4. Create ActivityService:
       - Inject ActivityRepository
       - log(data: CreateActivityDto): creates activity record
       - getByBoard(boardId, limit = 50): returns recent activities
       - getByEntity(entityType, entityId): returns entity's history
    
    5. Create ActivityController @Controller('activity'):
       - GET /board/:boardId - get board activities
       - GET /:entityType/:entityId - get entity history
    
    6. Create ActivityModule:
       - Provide AuditLogInterceptor as APP_INTERCEPTOR (global)
       - Export ActivityService
    
    7. Add @AuditLog decorators to existing controllers:
       - TasksController: create, update, delete methods
       - ListsController: create, update, delete, reorder methods
       - CommentsController: create, delete methods
    
    8. Import ActivityModule in AppModule
  </action>
  <verify>
    `cd apps/api && pnpm build` succeeds
    Creating a task generates an activity entry
    GET /activity/board/:id returns activity list
  </verify>
  <done>Decorator pattern working, activities logged automatically</done>
</task>
